# Fast-Profit Sales Scripts

Use these scripts as starting points. Keep each message specific to one observed problem and one deliverable.

## Universal First Message

Subject: quick {{artifact}} for {{company}}

Hi {{name}},

I noticed {{specific_observation}}.

I run a small productized service that turns the rows you already have into {{artifact}}. If you send a small sample, I can return a clean version within 24 hours.

Entry price is {{price}}. If helpful, reply with `sample` and I will send a tiny anonymized example before you decide.

Best,
{{your_name}}

## ReturnReply Pro

Observation ideas:

- Return requests mention the same reasons repeatedly.
- Store-credit or exchange paths are inconsistent.
- Support replies are being written one at a time.

Message:

Hi {{name}},

I noticed returns look like a margin and support workload problem for {{company}}. I can take 10-50 return request rows and return a decision queue, customer replies, and store-credit save paths within 24 hours.

The one-time setup is $149. If it saves support time, the monthly review is $199.

Worth sending a 10-row sample?

## AltText Cataloger

Observation ideas:

- Product grid has many images.
- New SKU drops are frequent.
- Catalog or SEO team is likely maintaining image metadata manually.

Message:

Hi {{name}},

I work on quick catalog cleanup projects. If you have product image rows, I can return a CSV with concise alt text, gap flags, and upload-ready copy for 100 images.

The first pass is $199. Ongoing new-SKU maintenance starts at $299/month.

## Chargeback Evidence Kit

Observation ideas:

- Merchant sells digital goods, subscriptions, or high-dispute products.
- Public community posts mention Stripe, Shopify, disputes, refunds, or fraud.
- Buyer likely has order records but no clean evidence packet.

Message:

Hi {{name}},

If your team has open disputes, I can organize one dispute into an evidence checklist, missing-item list, and counterargument draft before submission.

The packet is $249, or $349 for a rush turnaround. I do not promise win rates; the value is speed, completeness, and cleaner review.

Do you have one dispute worth using as a pilot?

## Follow-Up Cadence

Day 2:

Quick follow-up. Is {{artifact}} something your team already handles manually?

Day 5:

I can do this as a one-time cleanup before anything monthly. Want me to send the input format?

Day 10:

Should I close the loop, or check back next month when the queue is larger?

## Discovery Questions

- Where do these rows live today?
- How often does the queue renew?
- Who approves customer-facing copy?
- What would make the output immediately usable?
- What is the cost of not cleaning this up this month?
- If the first delivery works, who owns the monthly version?

## Objection Handling

| Objection | Response |
| --- | --- |
| We already do this internally | That is usually why the first pass works: I turn the manual steps into a cleaner queue your team can approve faster. |
| We need software, not a service | Agreed eventually. I recommend one service pass first so the software spec is based on real rows. |
| Too expensive | We can start with a smaller sample at a lower scope, then decide whether the full queue is worth it. |
| We cannot share data | Send anonymized rows with only the fields needed for the artifact. |
| Not this month | Should I check back after the next return/catalog/dispute batch? |
