# Revenue Ops SOP

This SOP turns the EM Fast-Profit portfolio into a repeatable selling motion.

## Tools

- Portfolio hub: `projects/index.html`
- Daily operating dashboard: `projects/monetization-command-center/src/index.html`
- Strategy: `docs/monetization-strategy.md`
- Scripts: `docs/sales-scripts.md`
- Launch playbook: `docs/launch-playbook.md`

## Daily Checklist

1. Pick one offer in the Monetization Command Center.
2. Set daily outreach target and expected conversion assumptions.
3. Export the current offer matrix if working with a partner or assistant.
4. Generate the outbound script.
5. Personalize and send 30 messages.
6. Log each prospect in a simple spreadsheet.
7. Deliver paid pilots with the matching static project tool.
8. Record delivery time and customer feedback.

## Paid Delivery Standard

Every paid delivery includes:

- Input snapshot.
- Tool-generated output.
- Human review.
- Three recommendations.
- Exported CSV or TXT.
- Short email summary.
- Maintenance upsell.

## Quality Review

Before sending work to a customer:

- Remove placeholder tokens.
- Check tone for customer-facing messages.
- Verify CSV output opens safely.
- Confirm no legal or compliance guarantee is implied.
- Add an approval note when the customer must review before use.

## Weekly Review

Every Friday:

1. Count outreach volume, replies, paid pilots, delivery time, and revenue.
2. Drop any offer below 3% reply rate after 60 targeted messages.
3. Keep any offer with one paid customer or two repeated requests.
4. Move repeated manual steps into the next product improvement backlog.
5. Refresh command center assumptions.

## Build Backlog Rules

Do not build from opinions. Build from paid friction.

| Signal | Action |
| --- | --- |
| Same input cleanup repeated 3 times | Add better import validation |
| Same export formatting request repeated 3 times | Add a template |
| Same customer buys twice | Offer monthly maintenance |
| Same agency asks twice | Add white-label package |
| Manual review takes over 60 minutes | Improve scoring or checklist |
